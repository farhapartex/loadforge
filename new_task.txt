  1. Core Engine — decouple from YAML files
                                                                                                                                                                                 
  - internal/loader/loader.go — currently accepts a file path. Change it to accept *config.Config directly so the web layer can pass an in-memory config without ever touching a
  file                                                                                                                                                                           
  - internal/config/config.go — split LoadConfig(path) into LoadConfig(path) (CLI keeps this) and DefaultConfig() (web uses this as a base to build on top of)
  - Remove the assumption that a scenario must come from disk — the engine should not care where the config came from                                                            
                                                                                                                                                                                 
  ---                                                                                                                                                                            
  2. New package — internal/openapi/                                                                                                                                             
                                                                                                                                                                                 
  This is the biggest new piece of work:
                                                                                                                                                                                 
  - fetcher.go — HTTP GET the spec URL, handle redirects, pass Authorization: Bearer <token> if token provided, detect content-type (JSON vs YAML)                               
  - parser.go — parse the raw spec into an internal representation; support both OpenAPI 3.x (openapi: 3.x.x) and Swagger 2.x (swagger: "2.0") — detect version automatically
  - extractor.go — walk the parsed spec and extract: base URL, all operations (method + path), path/query parameters, request body schema, security schemes                      
  - generator.go — convert extracted operations → *config.Config with one scenario per endpoint group (or one scenario with all steps), wire auth token as auth.bearer, apply    
  sensible defaults for load profile                                                                                                                                             
                                                                                                                                                                                 
  ---                                                                                                                                                                            
  3. New package — internal/runner/
                                   
  A bridge between the web layer and the load test engine:
                                                                                                                                                                                 
  - runner.go — wraps loader.Run(...), manages run lifecycle (start, stop, status), enforces only one active run at a time, writes progress to load_forge.logs                   
  - result.go — captures the final MetricsSnapshot when a run completes, stores it in memory for the History page                                                                
                                                                                                                                                                                 
  ---             
  4. Web backend — new endpoints                                                                                                                                                 
                                                                                                                                                                                 
  - web/handler_run.go — POST /api/run: validates inputs, calls openapi fetcher → parser → generator → runner; returns run ID immediately (non-blocking); 400 if a test is
  already running                                                                                                                                                                
  - web/handler_run.go — DELETE /api/run: stop the active test
  - web/handler_history.go — GET /history: render past runs from in-memory store
